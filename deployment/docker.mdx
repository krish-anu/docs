## Overview

Trash2Cash uses Docker Compose to orchestrate services including the backend (NestJS), frontend (Next.js), and PostgreSQL database.

## Development Setup

For local development, use `docker-compose.dev.yml`:

```bash
docker-compose -f docker-compose.dev.yml up --build
```

This starts:

- PostgreSQL database (port 5432) - optional, configured with `local-db` profile
- NestJS Backend (port 4000)
- Next.js Frontend (port 3000)

### Using Local PostgreSQL

To use a local PostgreSQL database during development:

```bash
docker-compose -f docker-compose.dev.yml --profile local-db up --build
```

Or configure an external database URL in `.env` and omit the `--profile` flag.

## Production Setup

For production, use `docker-compose.prod.yml` with prebuilt images:

```bash
docker-compose -f docker-compose.prod.yml up
```

This starts:

- Nginx reverse proxy (ports 80/443)
- NestJS Backend (exposed internally on port 4000)
- Next.js Frontend (exposed internally on port 3000)
- PostgreSQL (external managed database via DATABASE_URL)

## Docker Compose Configuration

### Development (docker-compose.dev.yml)

```yaml
services:
  # PostgreSQL (optional with local-db profile)
  db:
    image: postgres:16-alpine
    container_name: trash2cash-db
    environment:
      POSTGRES_USER: trash2cash
      POSTGRES_PASSWORD: trash2cash_dev_password
      POSTGRES_DB: trash2cash
    ports:
      - "5433:5432"
    profiles:
      - local-db

  # NestJS Backend
  backend:
    build:
      context: ./server
      dockerfile: Dockerfile
    ports:
      - "4000:4000"
    environment:
      DATABASE_URL: ${DATABASE_URL}
      DIRECT_URL: ${DIRECT_URL}
      PORT: 4000
      NODE_ENV: development
      JWT_ACCESS_SECRET: ${JWT_ACCESS_SECRET}
      JWT_REFRESH_SECRET: ${JWT_REFRESH_SECRET}
      CORS_ORIGIN: http://localhost:3000

  # Next.js Frontend
  frontend:
    build:
      context: ./client
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      NEXT_PUBLIC_API_URL: http://localhost:4000/api
```

### Production (docker-compose.prod.yml)

```yaml
services:
  # Nginx Reverse Proxy
  nginx:
    image: nginx:alpine
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"

  # NestJS Backend (prebuilt image)
  backend:
    image: anusan2003/trash2cash:backend
    restart: unless-stopped
    expose:
      - "4000"
    environment:
      DATABASE_URL: ${DATABASE_URL}
      DIRECT_URL: ${DIRECT_URL}
      NODE_ENV: production
      JWT_ACCESS_SECRET: ${JWT_ACCESS_SECRET}
      JWT_REFRESH_SECRET: ${JWT_REFRESH_SECRET}

  # Next.js Frontend (prebuilt image)
  frontend:
    image: anusan2003/trash2cash:frontend
    restart: unless-stopped
    expose:
      - "3000"
    environment:
      NEXT_PUBLIC_API_URL: ${NEXT_PUBLIC_API_URL}
```

## Building Docker Images

COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Production

FROM base AS production
ENV NODE_ENV=production

COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

EXPOSE 3000
CMD ["node", "server.js"]

````

## Running with Docker Compose

### Development Mode

```bash
# Start all services
docker compose up -d

# View logs
docker compose logs -f

# Stop services
docker compose down
````

### Production Mode

```bash
# Build for production
docker compose -f docker-compose.prod.yml build

# Start production
docker compose -f docker-compose.prod.yml up -d
```

## Database Operations

### Run Migrations

```bash
# Run migrations inside container
docker compose exec server npx prisma migrate deploy
```

### Seed Database

```bash
docker compose exec server npx prisma db seed
```

### Access Prisma Studio

```bash
docker compose exec server npx prisma studio
```

### Database Backup

```bash
# Backup
docker compose exec postgres pg_dump -U trash2cash trash2cash > backup.sql

# Restore
cat backup.sql | docker compose exec -T postgres psql -U trash2cash trash2cash
```

## Container Management

### View Running Containers

```bash
docker compose ps
```

### Restart a Service

```bash
docker compose restart server
```

### Rebuild a Service

```bash
docker compose build server
docker compose up -d server
```

### Clean Up

```bash
# Remove containers and volumes
docker compose down -v

# Remove images
docker compose down --rmi all
```

## Troubleshooting

### Check Service Health

```bash
docker compose ps
docker compose logs server
docker compose logs postgres
```

### Connect to Database

```bash
docker compose exec postgres psql -U trash2cash -d trash2cash
```
